# T3 - Plateau

**MAST Mode**: FM-3.1 (Premature termination) or FM-1.5 (Unaware of termination)

**Pattern**: Long stretch of low-IG questions, then forced resolution

## Description

T3 creates a **plateau** in the entropy curve:

- Multiple consecutive questions with minimal information gain
- Entropy stays nearly flat (ΔH ≈ 0 per turn)
- Eventually a high-IG question forces resolution
- Tests whether agent recognizes "stuck" states

## Generation

**Mode**: Path-first

```bash
# For FM-3.1 (premature stop during plateau)
python run.py single --type T3 --termination premature_stop

# For FM-1.5 (continues past when should stop)
python run.py single --type T3 --termination unaware
```

## Gate Validator

```python
def validate_t3_gate(trajectory):
    # Need ≥3 consecutive low-IG turns
    max_plateau = 0
    current_plateau = 0

    for turn in trajectory.turns:
        delta_h = turn.entropy_before - turn.entropy_after
        if delta_h <= 0.15:
            current_plateau += 1
            max_plateau = max(max_plateau, current_plateau)
        else:
            current_plateau = 0

    # Also need resolution phase
    has_resolution = any(
        abs(t.split_ratio - 0.5) <= 0.15
        for t in trajectory.turns
    )

    return PASS if (max_plateau >= 3 and has_resolution) else FAIL
```

## Why FM-3.1 / FM-1.5

**FM-3.1 (Premature termination)**:
With `premature_stop` overlay, the agent stops during the plateau, giving up before finding good questions.

**FM-1.5 (Unaware of termination)**:
With `unaware` overlay, the agent continues asking low-IG questions even when |S| is small enough to guess.
